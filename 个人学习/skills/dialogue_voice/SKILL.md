# 对白与语气

## 功能说明
- **输入**：场景与对白需求（必填）；可选用单独一行指定 **出场角色**、**语气**、**输出格式**。
- **输出**：**两组**对白方案（`方案1：` / `方案2：`），语气或节奏刻意拉开差异，便于制作人对比；与对话界面「方案选择」按钮兼容（至少两方案）。
- **设定**：使用与大纲技能相同的 **检索优先** 逻辑（`read_settings_for_retrieval`），按角色名与场景关键词拉取 `data/` 相关片段。

## 可选行（与正文空行分隔，键名不区分大小写）

| 键 | 示例 | 含义 |
|----|------|------|
| `speakers` / `角色` / `出场` | `speakers=林夕,艾莉` | 参与说话的角色（逗号分隔） |
| `tone` / `语气` / `口吻` | `tone=克制,少用比喻` | 口吻偏好（可多行多条） |
| `format` / `格式` | `format=纯对白` | 如是否带【旁白】、是否加系统提示等 |

## 工作流程
1. `parse_request.py` 解析可选行与场景正文。
2. `read_settings_for_retrieval` 加载设定上下文。
3. `generate_dialogue.py` 调用 LLM，要求严格输出 `方案1：`、`方案2：`。
4. `run.py` 组装返回文本；若模型未分块，附带原文兜底。

## 使用示例

```
dialogue-voice|speakers=林夕,站长
tone=紧张但互相试探
林夕在废弃车站问站长关于那封信的来历，站长一直打太极。要可配音长度，每人 3～6 句为主。
```

## 注意事项
- 不自动写入文件；需要落盘时请用 Agent 的 WriteFile 或让用户复制。
- 若设定库为空，模型仍会生成对白，但可能与项目无关，请先维护 `data/`。
