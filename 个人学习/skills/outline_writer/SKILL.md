# 大纲写手技能

## 功能说明

- **输入**：用户的故事构思（世界观、人物、主题、一句话剧情等均可）。
- **输出**：
  1. **分析与建议**：对构思的亮点、潜在问题、可优化方向进行点评，并提出 2～3 条具体建议。
  2. **多个大纲方案**：基于优化后的构思和本地设定集生成多个不同风格或走向的大纲方案（脚本默认 3 个，可通过 `generate_outlines` 的 `num_options` 调整为至多约 5 个，需与解析逻辑一致）。
- 支持根据用户选择的方案进一步细化或修改。

## 工作流程

### 步骤 1：分析点评

- 首先**不要调用任何工具**，仅根据用户提供的构思，输出「## 分析与建议」部分。
- 内容应包括：
  - **亮点**：构思中吸引人的地方（冲突、人物弧光、主题等）。
  - **潜在问题**：逻辑漏洞、动机不足、转折生硬等。
  - **优化建议**：2～3 条具体、可操作的改进方向。
- 输出格式示例：

```markdown
## 分析与建议

### 亮点
- ...

### 潜在问题
- ...

### 优化建议
1. ...
2. ...
```

### 步骤 2：读取设定（检索优先）

在输出分析点评后，调用 `read_settings.read_settings_for_outline`（内部为 `src.tools.setting_context.read_settings_for_retrieval`）。

- 使用与 SearchDocs 同源的分块索引，按剧情句关键词检索 `data/**/*.md` 并拼接相关分块；再附加截断后的广泛设定节选（`read_settings_bundle`）作兜底。
- 若检索无命中，则退回整包 `read_settings_bundle()`（与旧行为一致）。

读取到的设定内容作为生成大纲的上下文。

### 步骤 3：生成多个大纲方案

调用 `scripts/generate_outlines.py` 中的逻辑：将优化后的构思（可吸收用户对建议的反馈）、设定上下文与用户偏好（如「偏向悬疑」「注重感情线」）交给 LLM 生成多个方案。

每个方案需有清晰编号与若干段文字描述（约 200～300 字）。集成入口 `run.py` 会将结果展示为「方案1 / 方案2 / …」形式。

**推荐输出结构示例：**

```markdown
## 大纲方案

### 方案一：[标题]
[描述]

### 方案二：[标题]
[描述]
...
```

（若模型以编号列表 `1.` `2.` 输出，可与上述结构等价，由展示层整理为方案标题。）

### 步骤 4：用户选择与迭代细化

将方案以编号或 A/B/C 等形式提供给用户，提示选择（例如「请选择您喜欢的方案（输入 A/B/C 或编号）」）。

用户选择后，根据选中方案与修改意见再次调用 LLM，生成更详细的章节大纲或场景描述；如需落盘，使用 WriteFile 或由用户确认路径（可选）。

## 使用示例

**用户：**

我要写一个勇者背后的故事，村庄被魔王军威胁，村长被迫设陷阱……

**Agent（步骤 1 仅文本，示例节选）：**

```markdown
## 分析与建议

### 亮点
三幕结构清晰，村长牺牲的弧光完整。

### 潜在问题
村长转变动机略突兀；弟弟被杀可作转折但需更早铺垫。

### 优化建议
1. 增加村长与弟弟的亲情描写，让弟弟的死更具冲击力。
2. 明确魔王使者的人数或规则，解释为何不直接屠村而要用陷阱。
3. 为村民跳崖设计合理接应（如崖下河流或缓冲），避免让读者觉得降智。
```

（随后执行步骤 2、3：读取设定并生成多条大纲方案。）

## 注意事项

- 在步骤 1（分析点评）中**严禁调用任何工具**（含读取设定、专用生成脚本等），只输出文本分析。
- 步骤 2、3 中若需用户确认（如对优化建议是否接受），可短暂等待回复；默认可假设用户接受并继续，并提示：「将按上述建议生成大纲，如需调整请告知」。
