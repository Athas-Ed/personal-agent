# 设定版本管理技能

## 适用场景
当用户要求**修改** `data/` 目录下的任何设定文件时，必须使用本技能进行版本管理。  
当用户要求**回滚**或**清理历史版本**时，也通过本技能处理。

## 核心流程

### 1. 修改文件
**步骤：**
1. **备份原文件**  
   调用 `RunSkill`（`version-control|backup ...`）将原文件复制到 `{原文件所在目录}/历史版本/` 下。  
   备份文件名格式：`{原文件名}_{YYYYMMDD_HHMMSS}.{ext}`。  
   （例如：`角色设定/林夕.md` → `角色设定/历史版本/林夕_20250327_143022.md`）

2. **添加修改记录**  
   在原文件末尾（或紧接 `## 修改记录` 章节）追加一条记录，格式：
   - {时间}：{修改摘要}，备份：[备份文件名](历史版本/备份文件名)
- 若文件中不存在 `## 修改记录` 章节，则自动在文件末尾创建。
- 时间格式：`YYYY-MM-DD HH:MM:SS`
- 记录使用 Markdown 列表项，缩进为两个空格。

3. **应用修改**  
根据用户指示，使用 `WriteFile` 工具写入修改后的完整内容（或通过 `EditFile` 部分替换）。

### 2. 回滚文件
**步骤：**
1. **定位备份文件**  
- 若用户指定备份文件名（如 `林夕_20250327_143022.md`），直接使用。
- 若用户指定时间点（如“三天前”），则在原文件所在目录的 `历史版本/` 下查找最接近的备份文件（按时间戳解析）。

2. **恢复文件**  
调用 `CopyFile` 工具将备份文件内容覆盖到原文件。

3. **记录回滚**  
在原文件中添加一条修改记录，格式：
   {时间}：回滚到{备份文件名}，备份：[备份文件名]:/历史版本/{备份文件名}

### 3. 清理历史版本
当用户要求清理旧备份时（例如“清理半年之前的版本备份”），执行以下流程：

1. **调用清理函数**  
通过技能入口（`RunSkill`）或直接执行 `skills/version_control/scripts/version_manager.py` 中的 `clean_old_backups` 函数，传入目标根目录和保留天数。

2. **示例调用**  
ExecutePython("
from skills.version_control.scripts.version_manager import clean_old_backups
result = clean_old_backups('data/角色设定', older_than_days=180)
print(result)
")

3. **返回结果**  
将删除的文件列表和失败列表告知用户。

## 工具依赖
本技能依赖以下基础工具（已注册在 Agent 中）：
- `ReadFile`：读取文件（用于检查内容或恢复前的验证）
- `WriteFile`：写入文件（用于应用修改）

## 注意事项
- 所有备份文件必须放在原文件所在目录的 `历史版本` 子目录下，不允许跨目录存放。
- 备份文件的时间戳必须从文件名中解析（格式固定），以便后续清理和查找。
- 修改记录必须包含备份文件的相对路径（使用 Markdown 链接），确保可追溯。
- 如果文件在修改时已经存在修改记录章节，应追加记录，不能覆盖原有内容。
- 清理旧备份时，仅删除备份文件，不修改原文件（原文件中的记录保留，但备份链接可能失效——这由用户自行决定是否清理记录）。
