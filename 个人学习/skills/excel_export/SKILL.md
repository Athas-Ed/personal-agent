# Excel 导出

## 功能说明
- **输入**：模式 + `path=` 源 Markdown 相对路径 + 可选 `out=` 输出 `.xlsx` 路径。
- **输出**：写入 `data/exports/`（或你指定的相对路径），默认文件名为 `<源文件名>_export.xlsx`。
- **依赖**：`openpyxl`（版本在 `pyproject.toml` 中锁定）。

## 模式

| 模式 | 说明 |
|------|------|
| `dialogue` | 解析 `角色名：台词`（可带「」或引号），列：序号、角色、台词 |
| `task` | 解析 `##/### 标题` 小节，可选 `目标：`、`奖励：`；或 `- **任务名**` 列表 |
| `table` | 导出 Markdown **管道表格**（多表则多 Sheet：表1、表2…） |
| `both` | 同一文件同时导出「对话」「任务」两个工作表 |
| `all` | 对白 + 任务 + 文中全部表格 |

## 书写约定（源 Markdown）
- **对白**：一行一条，如 `林夕：「……」`、`艾莉: 嗯。`；以 `#` 开头的标题行会跳过；`注意：`、`说明：` 等不会当作角色。
- **任务**：每个任务用 `## 某任务名` 或 `###` 起头，正文中可写 `目标：…`、`奖励：…`。
- **表格**：标准 GFM：表头行 + `| --- | --- |` 分隔行 + 数据行。

## 示例

```
excel-export|dialogue
path=data/草稿/场景对白.md
out=data/exports/场景对白.xlsx
```

```
excel-export|table
path=data/策划/任务表.md
```

## 注意事项
- 仅允许**相对项目根**的路径，禁止 `..`。
- 若解析结果为空，会返回提示而不会覆盖生成空表（`all` 全无内容时除外）。
