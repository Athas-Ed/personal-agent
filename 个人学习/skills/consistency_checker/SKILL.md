# 设定一致性检查

## 功能说明
- 将 `data/` 下（默认 `角色设定/`、`背景设定/`）的 Markdown 汇总为上下文，调用 LLM 做**矛盾与疑点扫描**。
- 可选 **focus=** 关键词：先按关键词做分块检索（与 SearchDocs 同源），再附一段广泛节选，适合大库或只关心部分实体时省 token。
- 可选 **dirs=** / **scope=** / **范围=**：限定扫描的二级目录（逗号分隔），例如只查 `角色设定`。
- 其余自然语言作为**检查侧重点**（如「重点核对时间线」「只看艾莉家族关系」）。

## 工作流程
1. 解析用户输入中的 `focus` / `dirs` 等可选行，组装设定正文（见 `scripts/load_context.py`）。
2. 调用 `scripts/check_consistency.py` 生成审计提示词，**低温度**调用 LLM。
3. 返回带简短前缀的 Markdown 报告（摘要 + 有序列表疑点）。

## 使用示例

**全盘（默认两目录）**
```
setting-consistency|检查设定里有没有时间线或年龄对不上的地方
```

**缩小目录**
```
setting-consistency|dirs=角色设定
请重点看角色年龄和亲属关系
```

**关键词检索 + 侧重点**
```
setting-consistency|focus=林夕,废弃车站
重点核对她在不同文件中的出场设定是否冲突
```

## 注意事项
- 结论依赖模型与可见正文；正文过长会在约 10.5 万字符处截断，报告中会提示。
- 模型可能产生「弱疑点」，需人工对照原文确认；不会自动修改任何文件。
- 若 data 为空或未命中 focus，行为见运行结果中的说明。
