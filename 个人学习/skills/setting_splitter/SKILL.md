# 设定拆分器技能

## 功能说明
- 输入：一个包含多类设定的长文本（通常为 `.txt` 或 `.md` 的内容粘贴）
- 输出：按「人物 / 种族 / 势力 / 地点 / 历史」等类型拆分后的多个 Markdown 文件；**不增加用户未提供的情节、设定或评论**

## 执行原则（节省 Token、减少返工）
1. **先规划再落盘（适用于由 Agent 手动用 WriteFile 拆分时）**：若由模型逐文件写入，应先列出「条目名称 → 目标相对路径」再落盘。若使用本技能脚本 `run.py`，脚本已按下方「推荐大纲格式」解析，**不必**再让模型先输出规划表（避免多轮 JSON 与重复粘贴原文）。
2. **只搬运、不发散**：禁止「顺便」补充世界观、伏笔、人物小传或润色扩写；格式整理（标题、列表）允许，语义须与用户原文一致。
3. **清理遗留目录**：若本轮或历史拆分曾产生错误路径（见下方「已废弃目录名」），在确认目标文件已正确写入后，应删除错误目录下残留 `.md` 及**空文件夹**（可递归向上删除变为空的父目录）。

## 推荐大纲格式（脚本优先识别）
与下列结构一致时，拆分最稳定（`1．概述` 等为**条目内小节**，不会误当成独立文件）：
- 大节：`一、核心势力与冲突` / `二、关键地点` / `三、核心人物`（`一、` 与标题之间可无空格）
- 实体行：`（一）审判日组织` / `（一）莱茵生命生态科一号园区` / `（一）极青`（全角括号 + 中文或数字序号）
- 小节：`1．概述`（全角 `．` 或半角 `.` 均可）、`2．核心目标` …

**勿**在拆分完成后，把 `未分类/概述.md` 等误拆文件与正确实体文件一并删除；若脚本结果异常，应检查原文是否符合上述层级，或改用 `WriteFile` 按路径手动整理。

## 工作流程（脚本 `scripts/run.py` 与手动拆分均适用）
1. **读取与分割**：将输入文本按双换行符（`\n\n`）分割为段落块。
2. **分析与识别**：对每个段落块识别标题行与关键词（如 `##/###`、`人物：`、`种族：` 等），判断条目类型并提取条目名称；对「1．xxx / 2．yyy」编号列表会拆成多条独立条目。
3. **分类映射**：将类型映射到目录（见下方「目录映射」）。
4. **生成文件名**：使用条目名称清理非法字符生成文件名（追加 `.md`）。
5. **格式化内容**：输出为规范 Markdown：`## 条目名`，小节用 `### 小节标题`；自动将 Word 粘贴的制表符/特殊空白规范化，将常见项目符号（如 、•、· 等）转为 `- `；将独立行「键：值」整理为 `- **键**：值`；**去掉正文开头与 `##`/`###` 标题完全重复的一行**（如标题下再单独一行同名）。**不杜撰原文没有的正文**。
6. **保存文件**：在对应目录下**扁平**写入（不在类型目录下再套「集合名」子文件夹）。
7. **收尾**：输出本次写入路径清单；脚本 `run()` 会在**本次涉及的类型根目录**（如 `data/背景设定/势力/`）下自动**由深到浅删除空子文件夹**（不删除类型根目录本身，且仅允许白名单内的路径，不会扫整个 `data/`）。

## 目录映射（默认）
- `人物/角色` → `data/角色设定/`
- `种族` → `data/背景设定/种族/`
- `势力`（含用户文中「阵营」「派系」等组织向表述）→ `data/背景设定/势力/`
- `地点`（含「地图」「区域」等地理向表述）→ `data/背景设定/地点/`
- `历史` → `data/背景设定/历史/`
- 其他/无法识别 → `data/背景设定/未分类/`

### 已废弃目录名（迁移后应清理）
若仓库中仍存在以下仅因旧版规则生成的目录，且其中文件已迁走或属误拆产物，应删除其中残留并移除空目录：
- `data/背景设定/阵营/`（已由 `势力/` 替代）
- `data/背景设定/地图/`（已由 `地点/` 替代）

## 使用示例
输入片段：

## 人物：艾莉亚·星影
- 种族：高等精灵
- 阵营：银月联盟
- 职业：法师
- 描述：一位年轻的高等精灵法师，擅长星辰魔法，性格冷静。

## 种族：高等精灵
- 特征：尖耳、长寿、魔法亲和
- 分布：银光森林
- 历史：上古魔法文明的继承者。

输出文件（示例）：
- `data/角色设定/艾莉亚·星影.md`
- `data/背景设定/种族/高等精灵.md`
